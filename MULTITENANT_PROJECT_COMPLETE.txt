## ğŸ“ KEY DESIGN DECISIONS

1. **CompanyUsageDaily Table** (vs querying raw data)
   - Pro: OWNER cannot accidentally see employee details
   - Pro: Fast aggregates for dashboard
   - Pro: Clear separation of concerns

2. **X-Company-Key Header** (vs path param)
   - Pro: No ambiguity in routing
   - Pro: Validated by middleware before view
   - Pro: Cleaner API contracts

3. **Multiple Permission Classes** (vs single monolith)
   - Pro: Composable, testable
   - Pro: Easy to modify per-endpoint
   - Pro: Self-documenting code

4. **Subscription Model** (vs direct fields)
   - Pro: Audit trail of plan changes
   - Pro: Supports billing integration
   - Pro: Historical data preservation

5. **Null=True on New FKs** (vs default company)
   - Pro: Zero data loss during migration
   - Pro: Existing systems unaffected
   - Pro: Explicit migration path
